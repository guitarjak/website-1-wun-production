Design Principles

Consistency

Use the same colors, typography, spacing, and component patterns throughout the application to create a cohesive experience.

Clarity

Information should be easy to understand and navigate. Use clear labeling, hierarchy, and visual feedback.

Accessibility

Ensure sufficient color contrast, provide keyboard navigation, and use semantic HTML for screen readers.

Feedback

Provide visual feedback for user actions through animations, color changes, and status indicators.

Efficiency

Design for quick scanning and navigation. Minimize cognitive load and reduce unnecessary steps.

Delight

Include thoughtful micro-interactions and animations that make the interface feel polished and responsive.

Implementation Guide

How to use this design system in your Next.js React components.

CSS Variables

All colors and spacings are defined as CSS variables in the :root selector. Use them in your Tailwind config or directly in your CSS:

/* In your CSS files */
color: var(--primary-blue);
padding: var(--spacing-lg);
border-radius: var(--radius-md);

Component Library

Use the predefined Tailwind classes alongside this design guide:

  • Primary: bg-blue-600 for main actions
  • Success: bg-green-100 for positive feedback
  • Warning: bg-yellow-100 for caution
  • Error: bg-red-100 for destructive actions

Color Usage Guidelines

  • Primary Blue (#3B82F6): Main buttons, links, and primary actions
  • Primary Pink (#EC4899): Accent elements, highlights, and hover states
  • Success Green (#10B981): Completion badges, success messages, and positive feedback
  • Warning Yellow (#F59E0B): In-progress states and caution messages
  • Slate grays: Text, backgrounds, borders, and neutral elements
